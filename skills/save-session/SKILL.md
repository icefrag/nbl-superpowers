# Save Session Skill

将当前会话的所有内容——构建了什么、什么有效、什么失败、还有什么未完成——写入一个日期文件，以便下一次会话可以准确从离开的地方继续。

## 激活时机

- 结束工作会话，关闭 Claude Code 之前
- 达到上下文限制之前（先运行此命令，然后开始新会话）
- 解决了一个复杂的、希望记住的问题之后
- 任何需要将上下文传递给未来会话的时候

## 执行流程

### 步骤1：收集上下文

在写入文件之前，收集：

- 读取本次会话中修改的所有文件（使用 git diff 或从对话中回忆）
- 回顾讨论、尝试和决定的内容
- 记录遇到的任何错误以及如何解决（或未解决）
- 如果相关，检查当前的测试/构建状态

### 步骤2：创建 sessions 文件夹

```bash
mkdir -p ~/.claude/sessions
```

### 步骤3：写入会话文件

创建 `~/.claude/sessions/YYYY-MM-DD-<short-id>-session.tmp`，使用今天的实际日期和一个满足规则的 short-id：

- 允许的字符：小写 `a-z`，数字 `0-9`，连字符 `-`
- 最小长度：8 个字符
- 不允许大写字母、下划线、空格

有效示例：`abc123de`、`a1b2c3d4`、`frontend-worktree-1`
无效示例：`ABC123de`（大写）、`short`（少于8个字符）、`test_id1`（下划线）

### 步骤4：填充文件内容

诚实写入每个部分。不要跳过部分——如果某个部分确实没有内容，写"暂无"或"N/A"。

### 步骤5：向用户展示文件

写入后，显示完整内容并询问：

```
Session saved to [会话文件的实际路径]

这看起来准确吗？关闭前有什么需要修正或补充的吗？
```

等待确认。如请求则进行编辑。

## 会话文件格式

```markdown
# Session: YYYY-MM-DD

**Started:** [大致开始时间]
**Last Updated:** [当前时间]
**Project:** [项目名称或路径]
**Topic:** [本次会话主题的一行摘要]

---

## What We Are Building

[1-3段描述功能、bug修复或任务。包含足够的上下文，让对本次会话零记忆的人也能理解目标。]

---

## What WORKED (with evidence)

[仅列出已确认有效的内容。每项包含证明其有效的证据。]

- **[有效的功能]** — 确认方式：[具体证据]

如无确认有效的内容："暂无确认有效的内容——所有方法仍在进行中或未测试。"

---

## What Did NOT Work (and why)

[这是最重要的部分。列出每个尝试过但失败的方法。]

- **[尝试的方法]** — 失败原因：[确切原因/错误消息]

如无失败："暂无失败的方法。"

---

## What Has NOT Been Tried Yet

[有前景但未尝试的方法。对话中的想法。值得探索的替代方案。]

- [方法/想法]

如无待尝试项："暂无特定未尝试的方法。"

---

## Current State of Files

[本次会话触及的每个文件。精确描述每个文件的状态。]

| File              | Status         | Notes                      |
| ----------------- | -------------- | -------------------------- |
| `path/to/file.ts` | ✅ Complete    | [功能描述]                 |
| `path/to/file.ts` | 🔄 In Progress | [已完成，待完成]           |
| `path/to/file.ts` | ❌ Broken      | [问题]                     |
| `path/to/file.ts` | 🗒️ Not Started | [计划但未触及]             |

如未修改文件："本次会话未修改文件。"

---

## Decisions Made

[架构选择、接受的权衡、选择的方法及原因。]

- **[决策]** — 原因：[为什么选择而非替代方案]

如无重大决策："本次会话无重大决策。"

---

## Blockers & Open Questions

[任何未解决的问题。出现但未回答的问题。等待的外部依赖。]

- [阻塞项/开放问题]

如无："无活动阻塞项。"

---

## Exact Next Step

[如已知：恢复时要做的最重要的事情。足够精确，恢复时无需思考从哪里开始。]

[如未知："下一步未确定——在开始前查看'未尝试'和'阻塞项'部分以确定方向。"]

---

## Environment & Setup Notes

[仅在相关时填写——运行项目所需的命令、环境变量、需要运行的服务等。]
```

---

## 注意事项

- 每个会话有自己的文件——从不追加到之前会话的文件
- "什么无效"部分最关键——没有它，未来会话会盲目重试失败的方法
- 如果用户在会话中途（而非结束时）请求保存，保存已知信息并清楚标记进行中的项目
- 文件旨在在下一次会话开始时通过 `/resume-session` 读取
- 使用规范的全局会话存储：`~/.claude/sessions/`
- 新会话文件优先使用 short-id 文件名格式
